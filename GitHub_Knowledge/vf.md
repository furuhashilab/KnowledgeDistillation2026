# fc_creativeVideo

## 概要
古橋研究室の動画編集部（fc_creativeVideo）が運用するナレッジ共有・制作管理リポジトリである。Adobe Premiere Pro・After Effectsを用いた動画編集手法の蓄積や、大学AVスタジオの利用調整、BGM選定などチームの制作活動全般を扱っている。TEDxデザインプロジェクト等の動画制作に関連して立ち上げられたと推察される。

## 主な活動・成果
- Issue #2（Premiere Pro 編集方法）: パネル構成・ショートカットキー・カット・テロップ・明るさ調整・音楽追加・書き出しまでの編集ワークフローを詳細にドキュメント化。
- Issue #3（After Effects 編集操作方法）: パネル構成・各種エフェクト（keylight、Lumetriカラー、グリーンバック合成、フラクタルノイズ等）・キーフレーム・レンダリング手順を網羅的にまとめた知識記事として整備。
- Issue #4（AVスタジオ利用調整）: 青山学院大学B棟4F AVスタジオを9/12・9/15・10/18の3日間予約完了。学術情報課からの注意事項（鍵管理・感染症対策・空調手配等）も記録。
- Issue #5（撮影TIPS）: 撮影に関するノウハウをメンバーが共有（コメント3件）。
- Issue #6（BGM案）: 使用BGMの候補をメンバー間で議論（コメント6件）。

## 使用技術・ツール
- Adobe Premiere Pro（カット編集・テロップ・カラー補正・書き出し）
- Adobe After Effects（モーショングラフィックス・エフェクト・グリーンバック合成・レンダリング）
- Adobe Media Encoder（動画書き出し）
- GitHub Issues（ナレッジ管理・制作進行管理）

## 得られた知見
- 動画編集ノウハウ（Premiere Pro・After Effectsの操作手順）をGitHub Issuesにドキュメントとして蓄積することで、チームメンバーが参照可能な内部Wikiとして機能させられる。属人化しやすい映像制作スキルをIssue形式で共有・継承する手法は、学生チームの技術引き継ぎに有効。
- AVスタジオ予約は大学の学術情報課を通じた正式申請が必要で、使用申請書に授受サインが求められる。感染症対策（換気・除菌）や空調手配も事前確認が必須。

## 未解決事項・課題

- BGM案（Issue #6）および撮影TIPS（Issue #5）の本文が空欄で、コメント内の詳細が取得できていないため内容が把握しきれない。

# googleearthstudio

## 概要
Google Earth Studioを活用した映像制作のノウハウ（Know-How）を蓄積・共有するリポジトリ。Google Earth Studioで出力した連番JPEGファイルをFFmpegで動画（MP4）に変換する手順が中心的なトピックとなっている。リポジトリサイズ（約140MB）から、実際の画像・動画素材も含まれていると推察される。

## 主な活動・成果
- Google Earth Studioから書き出した連番JPEG（例：インドネシア・タンボラ山周辺）をFFmpegでMP4動画に変換する実行例をIssueに記録
- FFmpegのパターンマッチング挙動に関するトラブルシューティングを実施：デフォルトのグロブマッチが連番ファイルを認識しない問題を発見し、`-pattern_type none`オプションを使うことで解決
- 変換コマンドの最終形として `-r 30 -pattern_type none -i ファイル名_%03d.jpeg -vcodec libx264 -pix_fmt yuv420p -r 60` を確立

## 使用技術・ツール
- Google Earth Studio：地球規模の3D地図映像を生成するWebアプリ
- FFmpeg：動画変換・エンコードCLIツール（libx264コーデック、yuv420pピクセルフォーマット使用）
- JPEG連番ファイル：Google Earth Studioの出力フォーマット
- プログラミング言語：なし（language: null）

## 得られた知見
- Google Earth StudioからFFmpegで動画変換する際、デフォルトのファイルパターンマッチングでは連番JPEGが認識されないエラーが発生する。`-pattern_type none`を明示的に指定することで回避できる。
- 入力フレームレート（`-r 30`）と出力フレームレート（`-r 60`）を分けて指定することで、スムーズな動画出力が可能になる。
- この変換ワークフロー（Google Earth Studio → 連番JPEG → FFmpeg → MP4）は、地理的な3D映像コンテンツ制作の標準的な手順として他プロジェクトへも転用できる。

## 未解決事項・課題
Issue #1「実行例」がオープンのままクローズされておらず、最終的な成果物（動画）の完成・確認状況が不明。

- リポジトリ: https://github.com/furuhashilab/googleearthstudio

- 動画タイトルへの年号記載は再利用性を損なう場合があり、汎用性を高めたい場合は「年度」より「所属機関名」を前面に出す命名が有効である。
- 動画素材の管理はSSDや個人端末に頼らず、制作中から研究室共有ドライブへの早期アップロードを徹底することで、属人化リスクを下げられる。
