# fc_creativeVideo

## 概要
古橋研究室の動画編集部（fc_creativeVideo）が運用するナレッジ共有・制作管理リポジトリである。Adobe Premiere Pro・After Effectsを用いた動画編集手法の蓄積や、大学AVスタジオの利用調整、BGM選定などチームの制作活動全般を扱っている。TEDxデザインプロジェクト等の動画制作に関連して立ち上げられたと推察される。

## 主な活動・成果
- Issue #2（Premiere Pro 編集方法）: パネル構成・ショートカットキー・カット・テロップ・明るさ調整・音楽追加・書き出しまでの編集ワークフローを詳細にドキュメント化。
- Issue #3（After Effects 編集操作方法）: パネル構成・各種エフェクト（keylight、Lumetriカラー、グリーンバック合成、フラクタルノイズ等）・キーフレーム・レンダリング手順を網羅的にまとめた知識記事として整備。
- Issue #4（AVスタジオ利用調整）: 青山学院大学B棟4F AVスタジオを9/12・9/15・10/18の3日間予約完了。学術情報課からの注意事項（鍵管理・感染症対策・空調手配等）も記録。
- Issue #5（撮影TIPS）: 撮影に関するノウハウをメンバーが共有（コメント3件）。
- Issue #6（BGM案）: 使用BGMの候補をメンバー間で議論（コメント6件）。

## 使用技術・ツール
- Adobe Premiere Pro（カット編集・テロップ・カラー補正・書き出し）
- Adobe After Effects（モーショングラフィックス・エフェクト・グリーンバック合成・レンダリング）
- Adobe Media Encoder（動画書き出し）
- GitHub Issues（ナレッジ管理・制作進行管理）

## 得られた知見
- 動画編集ノウハウ（Premiere Pro・After Effectsの操作手順）をGitHub Issuesにドキュメントとして蓄積することで、チームメンバーが参照可能な内部Wikiとして機能させられる。属人化しやすい映像制作スキルをIssue形式で共有・継承する手法は、学生チームの技術引き継ぎに有効。
- AVスタジオ予約は大学の学術情報課を通じた正式申請が必要で、使用申請書に授受サインが求められる。感染症対策（換気・除菌）や空調手配も事前確認が必須。

## 未解決事項・課題
- 全Issueがopenのままであり、各制作物の完成・納品状況が不明。
- BGM案（Issue #6）および撮影TIPS（Issue #5）の本文が空欄で、コメント内の詳細が取得できていないため内容が把握しきれない。

# Hackathon_07

## 概要
古橋研究室の7月ハッカソンにおける映像制作プロジェクトの管理リポジトリ。ドローン映像・インタビュー撮影・動画編集を組み合わせた短編映像作品の制作を目的としており、複数の学生メンバーがIssuesで細かく役割分担しながらハッカソン形式で集中制作を行った。

## 主な活動・成果
- Issue #1「ED編集」・Issue #3「OP」：オープニング・エンディング映像の編集作業を分担
- Issue #2「Tracking Text」：映像へのトラッキングテキスト合成作業
- Issue #4「先生のインタビュー撮影」・Issue #8「各サークル代表インタビュー撮影」：7月5日4限から撮影を実施
- Issue #5「ドローン離陸映像撮影」：ドローンによる空撮素材の収集
- Issue #6「絵コンテ作成」：映像構成の絵コンテ制作（7コメントの活発な議論）
- Issue #7「インタビュー文字起こし」：撮影素材のテキスト化
- Issue #9「素材集め」・Issue #11「ドローンの映像など数秒のカットをいくつか用意」：素材収集・整理
- Issue #12「素材つなげ」：収集素材の編集・結合作業
- Issue #13「グラレコ」・Issue #14「BGM選曲」：グラフィックレコーディングとBGM選曲（各3コメント）
- Issue #15「最終調整・修正」・Issue #18「発表用スライド」：完成直前の仕上げ作業
- Issue #19・#20「medium」・Issue #21「先輩のドローン映像作成」：Medium記事執筆と先輩ドローン映像の活用

## 使用技術・ツール
- ドローン（空撮映像素材の撮影）
- 動画編集ソフト（After Effects / Premiere等、推定）
- Medium（成果報告記事の公開）
- GitHub Issues（タスク管理・役割分担）
- グラフィックレコーディング

## 得られた知見
- ハッカソン形式の映像制作では、GitHub Issuesを工程管理ツールとして活用することで、撮影・編集・文字起こし・BGM・グラレコ・Medium執筆といった多岐にわたる作業を複数名で並行して進められる。絵コンテIsssue（#6）に7コメントが集中していることから、映像構成の議論がプロジェクトの重要なハブになることが確認できる。

## 未解決事項・課題
20件のIssueがすべてオープンのまま残っており、ハッカソン終了後にクローズ処理が行われなかった。成果物の完成状態はMedium等の外部サービスで公開されたと推定される。

# ShibuyaUrarojiStreetView

## 概要
渋谷の裏路地をテーマにしたストリートビュー撮影・公開プロジェクトのリポジトリ。Google ストリートビューや Mapillary 等のプラットフォームに掲載されにくい渋谷の路地裏・横丁などを360度カメラで撮影・記録し、地理空間情報として共有することを目的としている。古橋研究室のV&Fフィールドワーク活動の一環と考えられる。

## 主な活動・成果
- Issue #1「screenshots」にて撮影現場・活動の様子を示す写真（街頭での撮影シーン）が共有・記録された
- リポジトリ作成直後の短期間（2021年9月末〜10月初旬）に集中して活動が行われた
- リポジトリサイズが非常に小さく（3KB）、主に記録・告知目的での運用

## 使用技術・ツール
- 360度カメラ（ストリートビュー撮影用）
- Mapillary または Google ストリートビュー（撮影データの公開先、推定）
- GitHub Issues（活動記録・スクリーンショット共有）

## 得られた知見
- 渋谷の裏路地のような既存の地図サービスにカバーされていないエリアを、研究室のフィールドワークとして360度撮影・公開することで、OSMや Mapillary のオープンデータ整備に貢献できる。小規模なフィールド活動でもGitHubリポジトリを起点に記録を残すことで、後続の参照・継続が容易になる。

## 未解決事項・課題
Issue #1「screenshots」が1件オープンのままクローズされていない。撮影成果物の公開先URLや最終的なデータ整備状況は本リポジトリ上では確認できない。
