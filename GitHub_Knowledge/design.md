## 主な活動・成果
- kawagoe を使ったベクタータイル＋ラスター画像オーバーレイを試みたが、構造理解が追いつかず作業を一時中断

# city-roads

# areamap

## 概要
本リポジトリは、日本の都道府県を区分ごとに色分けして表示するカラム地図（TabularMaps）を提供する tabularmaps/areamap を古橋研究室が fork したものである。統計データと地図を組み合わせたコロプレス表現を HTML ベースで実装しており、データの地域別可視化に活用できる。GitHub Pages によるウェブ公開にも対応しており、研究室内から 2 件の fork が派生している。

## 主な活動・成果
- tabularmaps/areamap を研究室組織アカウントに fork し、GitHub Pages で公開
- Issues は無効化されており、研究室独自の改変・議論の記録はない
- 研究室メンバーが都道府県別データ可視化の教材・参照実装として利用したとみられる
- fork 後約 1 日でプッシュが完了し、以降の更新はない

## 使用技術・ツール
- HTML（主要言語）
- TabularMaps（都道府県カラム地図ライブラリ）
- GitHub Pages（静的サイトホスティング）
- Git / GitHub（バージョン管理・fork による管理）

## 得られた知見
- TabularMaps のようなカラム地図手法は、都道府県単位の統計データを地理的文脈で直感的に表現するのに有効であり、地理的な面積の大小に引きずられない均等表示が可能である。HTML と JavaScript のみで実装できるため、GIS ソフトウェア不要で軽量なデータ可視化が実現できる点も利点である。

# tokyo-vector-tile

## 概要
本リポジトリは、東京都のオープンデータを活用したベクタータイル配信・可視化の実装を研究するために、smellman/tokyo-vector-tile を古橋研究室が fork したものである。東京都庁（Tokyo-Metro-Gov）も同リポジトリを fork していることから、行政と研究機関が連携する形でベクタータイル技術の普及・活用が図られた文脈がうかがえる。リポジトリのサイズが約 550MB と大きく、実際のタイルデータを含む形で管理されていたとみられる。

## 主な活動・成果
- smellman/tokyo-vector-tile を研究室アカウントに fork し、東京都ベクタータイルの実装を取り込み
- Issues は無効化されており、研究室独自の改変・議論の記録はない
- fork 元（smellman 氏）のリポジトリは 2020年3月に作成され、東京都庁アカウントにも fork されるなど行政オープンデータとベクタータイル技術の接点として注目度が高い
- 研究室での利用は参照・教育目的とみられる

## 使用技術・ツール
- HTML（fork 元の主要言語）
- Vector Tile（MVT: Mapbox Vector Tile 形式）
- MapLibre GL JS / Mapbox GL JS（ベクタータイル描画ライブラリ）
- 東京都オープンデータ（地図データソース）
- GitHub Pages（タイルビューワのホスティング）
- Git / GitHub（バージョン管理・fork による管理）

## 得られた知見
- 行政機関が GitHub でベクタータイルデータを公開・管理する事例として東京都は先駆的であり、研究・教育機関が fork するだけで最新の都市データを地図可視化の教材として活用できる。ただし大容量のタイルデータを Git リポジトリに直接含める運用は管理コストが高く、CDN や専用タイルサーバへの分離を検討すべきである。
